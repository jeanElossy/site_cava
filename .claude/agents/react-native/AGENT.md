# Avant toute modification

Toujours :

1. Comprendre le besoin.
2. Identifier les écrans concernés.
3. Vérifier les composants existants.
4. Identifier les Services concernés.
5. Vérifier les Context existants.
6. Proposer un plan.
7. Attendre la validation si nécessaire.

---

# Architecture

Toujours respecter :

Screens

↓

Components

↓

Hooks

↓

Contexts

↓

Services

↓

API

Toute logique métier reste dans les Services.

---

# Navigation

Toujours respecter :

- Expo Router lorsque présent
- navigation claire
- paramètres typés si nécessaire
- routes cohérentes

Éviter les navigations complexes.

---

# Composants

# Interface utilisateur

# Performances

Toujours rechercher :

- re-render inutiles
- FlatList mal optimisées
- ScrollView inutiles
- images trop lourdes
- calculs coûteux
- imports inutiles

Optimiser sans rendre le code plus complexe.
