# Développement d'une nouvelle fonctionnalité

## Objectif

Développer une nouvelle fonctionnalité de manière professionnelle.

Tu ne dois jamais commencer à coder immédiatement.

Tu dois d'abord comprendre le besoin, analyser l'architecture existante, proposer une solution adaptée puis implémenter la fonctionnalité en respectant les standards du projet.

---

# Workflow obligatoire

Toujours respecter cet ordre.

---

## 1. Comprendre le besoin

Identifier :

- l'objectif métier
- les utilisateurs concernés
- le résultat attendu
- les contraintes éventuelles

Si une information manque, poser les questions nécessaires.

---

## 2. Analyser l'existant

Identifier :

- composants existants
- services existants
- hooks existants
- contextes existants
- modèles existants
- API existantes

Toujours privilégier la réutilisation.

Ne jamais recréer ce qui existe déjà.

---

## 3. Consulter les agents concernés

Selon la fonctionnalité, consulter :

- Architect Agent
- Frontend Agent
- React Native Agent
- Backend Agent
- Database Agent
- Security Agent
- Performance Agent
- UX/UI Agent
- QA Agent
- Documentation Agent
- SEO Agent (Web uniquement)
- DevOps Agent (si nécessaire)

Leurs recommandations doivent être prises en compte avant l'implémentation.

## 8. Implémenter

Pendant le développement :

Toujours respecter :

- l'architecture du projet
- les Coding Standards
- les conventions de nommage
- les bonnes pratiques
- la sécurité
- les performances

Ne jamais :

- dupliquer du code
- casser une fonctionnalité existante
- ignorer les erreurs
- contourner l'architecture

---

## 9. Vérifier

Après l'implémentation :

Toujours contrôler :

- fonctionnement
- sécurité
- performances
- responsive (si interface)
- accessibilité (si interface)
- qualité du code
- absence de régression

---

- cas nominal

- cas limites
- validation des données
- gestion des erreurs

# Philosophie

Chaque nouvelle fonctionnalité doit améliorer durablement le projet.

Elle doit être :

- utile
- propre
- réutilisable
- testable
- maintenable
- sécurisée
- performante

Ne jamais privilégier la rapidité au détriment de la qualité.
