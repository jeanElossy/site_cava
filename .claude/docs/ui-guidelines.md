# UI / UX Guidelines

## Objectif

Toutes les interfaces doivent être modernes, élégantes, cohérentes et intuitives.

L'expérience utilisateur est aussi importante que les fonctionnalités.

---

# Philosophie

Toujours privilégier :

- la simplicité
- la lisibilité
- la cohérence
- la rapidité
- l'accessibilité

Une interface doit être belle sans être surchargée.

---

# Design

Le style général doit être :

- moderne
- professionnel
- épuré
- responsive
- accessible

Éviter :

- les interfaces surchargées
- les couleurs agressives
- les animations inutiles

---

# Couleurs

Toujours respecter la palette du projet.

Les couleurs principales doivent être définies dans un seul endroit.

Éviter les couleurs codées directement dans les composants.

Utiliser :

theme.js

ou

constants/colors.js

---

# Typographie

Toujours respecter une hiérarchie.

Exemple :

H1

H2

H3

Body

Caption

Limiter le nombre de polices utilisées.

---

# Espacement

Toujours utiliser un système cohérent.

Exemple :

4px

8px

12px

16px

24px

32px

48px

Éviter les valeurs aléatoires.

---

# Boutons

Tous les boutons doivent :

- avoir un état normal
- un état hover (web)
- un état actif
- un état désactivé
- un état de chargement

Les boutons importants doivent être clairement visibles.

---

# Cartes

Les cartes doivent être :

- aérées
- lisibles
- cohérentes

Éviter les bordures trop épaisses.

Privilégier :

- coins arrondis
- ombres légères
- espacements réguliers

---

# Formulaires

Toujours :

- afficher les erreurs clairement
- afficher un label
- afficher un placeholder pertinent
- indiquer les champs obligatoires

Ne jamais masquer les erreurs.

---

# Icônes

Utiliser une seule bibliothèque d'icônes.

Les icônes doivent avoir une taille cohérente.

Éviter de mélanger plusieurs styles d'icônes.

---

# Responsive

Le site doit fonctionner correctement sur :

- téléphone
- tablette
- ordinateur

Ne jamais concevoir uniquement pour desktop.

---

# Animations

Les animations doivent :

- être fluides
- être rapides
- améliorer l'expérience

Éviter :

- les animations longues
- les effets distrayants

---

# Accessibilité

Toujours respecter :

- contraste suffisant
- navigation clavier (web)
- textes alternatifs pour les images
- structure HTML correcte

---

# Performance

Toujours :

- optimiser les images
- limiter les animations coûteuses
- éviter les composants inutiles

---

# Cohérence

Tous les écrans doivent respecter :

- les mêmes couleurs
- les mêmes marges
- les mêmes boutons
- les mêmes cartes
- les mêmes formulaires

Le projet doit donner l'impression d'avoir été conçu par un seul designer.

---

# Avant chaque livraison

Toujours vérifier :

☐ Interface responsive

☐ Aucun débordement

☐ Contrastes suffisants

☐ Boutons cohérents

☐ Espacements réguliers

☐ Icônes homogènes

☐ Animations fluides

☐ Bonne lisibilité

☐ Expérience utilisateur agréable

---

# Philosophie finale

Une bonne interface doit être :

- simple
- élégante
- intuitive
- rapide
- cohérente

L'utilisateur ne doit jamais avoir à réfléchir pour comprendre comment utiliser l'application.
