# Conventions du projet

## Objectif

Ce document contient les conventions permanentes du projet.

Claude doit toujours respecter ces conventions avant de proposer ou modifier du code.

---

# Langage

Le projet utilise principalement :

- JavaScript
- JSX

Ne jamais convertir le projet en TypeScript sans autorisation.

---

# Frontend

Technologies :

- React
- React Native
- Expo (pour le mobile)

Toujours privilégier :

- composants fonctionnels
- Hooks
- composants réutilisables
- architecture modulaire

---

# Backend

Technologies :

- Node.js
- Express
- MongoDB
- Mongoose

Toujours respecter :

Routes
↓

Controllers
↓

Services
↓

Models

---

# Style de développement

Toujours écrire :

- un code lisible
- un code maintenable
- un code performant

Éviter la duplication.

---

# Dépendances

Avant d'installer une nouvelle bibliothèque :

- expliquer son utilité
- proposer une alternative si possible
- demander une validation

---

# Structure

Respecter l'organisation actuelle.

Ne jamais déplacer massivement des fichiers sans validation.

---

# Communication

Toutes les explications sont en français.

Le code reste en anglais lorsque c'est la convention.

---

# Objectif

Produire un code de qualité professionnelle, cohérent avec l'ensemble du projet.
