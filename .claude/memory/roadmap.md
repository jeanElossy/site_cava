# Roadmap du projet

## Objectif

Ce document décrit la vision du projet ainsi que les fonctionnalités prévues.

Avant de proposer une nouvelle fonctionnalité ou une modification importante, Claude doit vérifier qu'elle reste compatible avec cette roadmap.

---

# Vision

Le projet doit être :

- Professionnel
- Moderne
- Rapide
- Sécurisé
- Évolutif
- Facile à maintenir

Toutes les décisions doivent aller dans cette direction.

---

# Priorités

Toujours privilégier :

1. Qualité
2. Sécurité
3. Performance
4. Expérience utilisateur
5. Évolutivité
6. Maintenabilité

Toujours vérifier :

- qu'elle respecte l'architecture

# Objectif final

Construire un logiciel :

- fiable
- robuste
- sécurisé
- évolutif
- agréable à utiliser

Chaque nouvelle fonctionnalité doit rapprocher le projet de cette vision.
